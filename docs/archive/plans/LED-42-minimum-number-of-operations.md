# LED-42: Minimum Number of Operations

- Status: Completed
- Date: 2026-06-15
- Jira: https://gorushkin.atlassian.net/browse/LED-42
- PR: TBD

## Цель

Гарантировать доменный инвариант: активная транзакция должна содержать не
менее двух операций.

## Границы

- Проверять инвариант при создании новой транзакции.
- Проверять результирующее количество активных операций до применения update.
- Не добавлять проверку в `Transaction.restore()`: восстановление загружает
  persistence snapshot и не должно частично конструировать агрегат.
- Не смешивать этот инвариант с балансом, количеством уникальных счетов и
  другими задачами эпика `LED-40`.

## Критерии завершения

- Создание транзакции с `0` или `1` операцией завершается
  `InsufficientOperationsError`.
- Создание транзакции с двумя или более операциями переходит к остальным
  доменным проверкам.
- Update, результатом которого станет меньше двух активных операций,
  отклоняется до мутации агрегата.
- Ошибка update не меняет snapshot, version и timestamps.
- Все затронутые проверки проходят.
